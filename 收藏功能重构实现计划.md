# Radar 收藏功能重构实现与技术计划

## 1. 背景与目标

当前 `agents-radar` 已有一套“收藏到 SecondBrain”的功能，但它更像摘录归档：

- 网站端在链接旁和段落旁插入 `☆` 按钮。
- 点击后通过本机 `Radar Bridge` 写入 SecondBrain。
- Bridge 目前会为每条收藏生成单独 Markdown 文件，路径类似 `收藏/2026/2026-06/...md`。

这不符合新的产品目标。新的收藏功能应更像“网站内书签系统”：

- 收藏是为了统一回看有价值内容，不希望每天拆成不同文件夹或不同文件。
- 支持收藏三类对象：整篇报告、链接、段落摘录。
- 网站左侧有“我的收藏”入口，并可按类型筛选。
- 收藏列表按“收藏日期”倒序排列，来源日报日期只作为辅助信息展示。
- 收藏项可跳回原日报位置；链接收藏还要保留打开原始外链能力。
- 收藏状态在原文中有高亮提示。
- 收藏数据同时保存到 GitHub 和 SecondBrain。

## 2. 产品设计

### 2.1 收藏入口

左侧栏新增固定区域：

```text
★ 我的收藏
  全部收藏
  整篇报告
  链接
  段落
```

这不是文件夹结构，只是同一个收藏库的筛选视图。

底层只维护统一收藏数据，例如：

```text
favorites.json
```

SecondBrain 侧也只生成统一 Markdown，例如：

```text
收藏/AI情报收藏.md
```

### 2.2 时间规则

收藏系统以“收藏日期”为主：

- 收藏列表按 `created_at` 倒序。
- UI 分组建议为“今天收藏 / 昨天收藏 / 更早”或按日期标题分组。
- 原日报日期 `source.date` 只显示为来源信息。

示例：

```text
今天收藏

[链接] OpenClaw provider 兼容性讨论
来源：2026-06-10 · AI Agents 生态
收藏时间：2026-06-10 15:42
回到原文 · 打开外链

[段落] 关于 GitHub Trending 的趋势判断
来源：2026-06-09 · GitHub AI 趋势
收藏时间：2026-06-10 15:31
回到原文
```

### 2.3 三类收藏对象

#### 整篇报告

- 收藏按钮放在报告开头标题旁。
- 未收藏显示空心星 `☆`，已收藏显示实心星 `★`。
- 收藏后标题区域高亮。
- 收藏列表中只保存标题和来源，点击“回到报告”跳转到对应日报。

#### 链接

不要在每个链接后常驻星星，避免视觉污染。

推荐交互：

- 鼠标悬停链接时显示轻量浮层：`收藏链接`。
- 或右键链接时显示自定义菜单：`收藏这个链接`。
- 收藏后链接本身高亮，例如浅背景、虚线下划线或左侧小标记。

收藏列表中提供两个动作：

- `回到原文`：跳回日报中该链接所在位置，并临时高亮。
- `打开外链`：打开原始链接。

网站不调用 AI 识别链接含义。来源网站直接用 URL hostname 显示，例如 `github.com`、`arxiv.org`、`producthunt.com`。

#### 段落

不要常驻按钮。

推荐交互：

- 用户选中一段文字后，选区附近出现浮层：`收藏段落`。
- 或右键选区时出现自定义菜单：`收藏选中段落`。
- 收藏后原文对应段落高亮。
- 收藏列表展示完整摘录内容。
- 点击“回到原文”跳回该段落位置并临时闪烁高亮。

## 3. 数据与存储设计

### 3.1 GitHub 作为主存储

由于当前用户接受暂时公开，收藏数据可以提交到 GitHub Pages 仓库中，由网站直接读取。

建议新增根目录文件：

```text
favorites.json
```

建议结构：

```json
{
  "version": 1,
  "updated_at": "2026-06-10T08:00:00.000Z",
  "items": [
    {
      "id": "sha256...",
      "kind": "link",
      "title": "OpenClaw provider compatibility",
      "url": "https://github.com/example/repo/issues/1",
      "site": "github.com",
      "excerpt": "收藏时的上下文文本",
      "section": "Provider 兼容性",
      "created_at": "2026-06-10T07:42:00.000Z",
      "source": {
        "date": "2026-06-10",
        "report": "ai-agents",
        "label": "AI Agents 生态",
        "url": "https://ivo-eu.github.io/agents-radar/#2026-06-10/ai-agents",
        "anchor": "favorite-target-id"
      }
    }
  ]
}
```

字段约定：

- `id`：稳定 ID，用于去重和高亮。按收藏类型、来源、URL 或摘录内容生成。
- `kind`：`report`、`link`、`excerpt`。
- `created_at`：收藏时间，是列表排序主依据。
- `source.date`：日报生成日期，仅用于来源展示和跳转。
- `source.anchor`：原文定位锚点，用于回跳和高亮。
- `excerpt`：链接收藏保存上下文；段落收藏保存选中文本；整篇报告可为空或保存摘要。

### 3.2 写入 GitHub 的方式

不能把 GitHub Token 放进 GitHub Pages 前端。正确方式：

```text
Radar 网站 → 本机 Radar Bridge → 更新 favorites.json → git commit/push
```

Bridge 负责：

- 读取当前仓库的 `favorites.json`。
- 合并新增收藏。
- 去重。
- 写回文件。
- 执行 `git add favorites.json && git commit && git push`。

如果没有 Bridge 或 push 失败：

- 网站应仍然可以把收藏先保存到浏览器 `localStorage` 作为待同步队列。
- 顶部显示“有 N 条收藏未同步”。
- 用户连接 Bridge 后再同步到 GitHub。

### 3.3 SecondBrain 同步

SecondBrain 不再每条收藏一个文件。

建议统一生成：

```text
收藏/AI情报收藏.md
```

内容按收藏日期倒序：

```md
# AI 情报收藏

## 2026-06-10

### [链接] OpenClaw provider compatibility

- 网站：github.com
- 来源：2026-06-10 · AI Agents 生态
- 回到 Radar：<https://ivo-eu.github.io/agents-radar/?focus=xxx#2026-06-10/ai-agents>
- 原始链接：<https://github.com/example/repo/issues/1>

收藏时上下文：
> ...

### [段落] Provider 兼容性

- 来源：2026-06-10 · AI Agents 生态
- 回到 Radar：<https://ivo-eu.github.io/agents-radar/?focus=yyy#2026-06-10/ai-agents>

> 收藏的段落内容
```

SecondBrain 里不需要再实现复杂按钮或交互。它是收藏结果的长期沉淀视图，主要操作入口仍是 Radar 网站。

## 4. 技术实现计划

### 4.1 前端改造：`index.html`

保留现有报告浏览、搜索、主题切换能力，重写收藏相关逻辑。

需要新增或调整：

- 加载 `favorites.json`，不存在时按空收藏库处理。
- 左侧栏渲染“我的收藏”筛选入口。
- 新增收藏列表视图，和日报视图共用内容区域。
- 报告渲染后为标题、链接、段落生成稳定 `data-favorite-id`。
- 收藏过的目标添加高亮 class。
- 处理 `?focus=<id>#date/report`，加载报告后滚动到目标并临时高亮。
- 整篇报告标题旁显示收藏按钮。
- 链接收藏改为 hover 或右键浮层，不再常驻星星。
- 段落收藏改为选中文本后浮层或右键菜单。
- 未连接 Bridge 时先写 `localStorage` 待同步队列。
- 已连接 Bridge 时调用 Bridge API 同步到 GitHub 和 SecondBrain。

建议新增本地状态：

```js
let favorites = { version: 1, updated_at: '', items: [] };
let pendingFavorites = JSON.parse(localStorage.getItem('radar-pending-favorites') || '[]');
```

### 4.2 Bridge 改造：`bridge/cmd/radar-bridge/main.go`

现有 Bridge 已有配对、同步日报、收藏写入接口，但收藏模型要升级。

需要调整：

- 新增或改造 `/api/favorites`：
  - `GET`：返回当前 `favorites.json`。
  - `POST`：新增或更新一条收藏，返回收藏状态。
  - 可选 `DELETE`：取消收藏。
- 新增 `/api/favorites/sync`：
  - 合并浏览器待同步收藏。
  - 更新仓库根目录 `favorites.json`。
  - 生成或更新 SecondBrain 的 `收藏/AI情报收藏.md`。
  - commit + push 到 `origin/master`。
- 旧的“每条收藏生成单独 md 文件”逻辑不要继续作为默认行为。
- 保留安全限制：
  - 只监听 `127.0.0.1:4399`。
  - 继续使用 pairing token。
  - 不读取 SecondBrain 其他内容。
  - 不上传 SecondBrain 内容。

### 4.3 数据合并与去重规则

稳定 ID 规则：

- `report`：`report\n<date>\n<report>`
- `link`：`link\n<normalized_url>`
- `excerpt`：`excerpt\n<date>\n<report>\n<normalized_excerpt>`

合并规则：

- 同 ID 收藏只保留一条。
- 如果本地和远端同 ID 都存在，保留较早的 `created_at`，但更新缺失字段。
- 列表排序始终按 `created_at` 降序。
- 取消收藏如果 v1 要做，建议使用 `deleted_at` 软删除，避免多端合并时反复出现。

### 4.4 GitHub Pages 发布流程

正式网站应使用：

```text
https://ivo-eu.github.io/agents-radar
```

本地网站只作为预览：

```text
http://127.0.0.1:4173
```

收藏功能以 GitHub Pages 为主目标设计。Bridge 配对时要求 return URL 与 `pages_url` 一致，因此本地预览如果也要测试收藏，需要额外支持 localhost 白名单或使用 Pages 测试。

建议 v1：

- 正式收藏只在 GitHub Pages 上启用。
- 本地预览可读收藏，但写入时提示“请在正式 Pages 网站使用收藏同步”。

## 5. 测试与验收

### 5.1 前端手动测试

- 打开 GitHub Pages，确认左侧出现“我的收藏”。
- 收藏整篇报告后：
  - 标题旁星星变实心。
  - 标题区域高亮。
  - 收藏列表出现该报告。
  - 点击“回到报告”能跳回原日报。
- 收藏链接后：
  - 链接本身高亮。
  - 收藏列表出现来源网站 hostname。
  - `回到原文` 跳回日报位置。
  - `打开外链` 打开原始链接。
- 收藏段落后：
  - 段落高亮。
  - 收藏列表展示完整摘录。
  - 点击“回到原文”能滚动并闪烁高亮。
- 刷新页面后收藏状态仍存在。
- 换到另一个浏览器打开 Pages，能通过 `favorites.json` 看到已同步收藏。

### 5.2 Bridge 测试

- `radar-bridge serve` 后 `/api/status` 正常。
- 新增收藏会更新仓库根目录 `favorites.json`。
- 新增收藏会更新 SecondBrain 的 `收藏/AI情报收藏.md`。
- 重复收藏同一链接不会产生重复条目。
- Bridge 不在线时，网站进入待同步状态，不丢收藏。
- Bridge 恢复后，待同步收藏能合并进 GitHub 和 SecondBrain。

### 5.3 自动化测试

建议新增：

- JS 层收藏 ID 生成和去重测试。
- Bridge 层收藏合并测试。
- Bridge 层 SecondBrain Markdown 渲染测试。
- Bridge 层 git 失败时的错误处理测试。
- 现有 `pnpm test`、`pnpm typecheck` 保持通过。
- Bridge 目录 `go test ./...` 保持通过。

## 6. 迁移策略

当前已有旧收藏逻辑可以直接替换，但为了避免丢数据：

- 如果 Bridge state 里已有旧 `Favorites` 记录，可以先不迁移。
- 新功能上线后只写新的 `favorites.json` 和统一 SecondBrain 文件。
- 旧的单条收藏 Markdown 文件保留不动，视为历史数据。

## 7. 分阶段实施建议

### Phase 1：网站书签体验

- 新增 `favorites.json` 读取。
- 新增左侧“我的收藏”视图。
- 实现整篇报告、链接、段落三类收藏 UI。
- 收藏先写入 `localStorage`，完成高亮和回跳体验。

### Phase 2：Bridge 写入 GitHub

- 扩展 Bridge 收藏 API。
- 合并 `localStorage` 待同步收藏到 `favorites.json`。
- 支持 git commit/push。
- Pages 刷新后能读取公开收藏。

### Phase 3：SecondBrain 统一收藏库

- Bridge 生成 `收藏/AI情报收藏.md`。
- 按收藏日期倒序写入。
- 每条收藏包含回到 Radar 的链接。

### Phase 4：打磨

- 取消收藏。
- 收藏搜索。
- 标签或备注。
- 按来源日期筛选。
- 本地预览环境收藏测试支持。

## 9. 当前明确决策

- 主排序使用收藏日期，不使用日报生成日期。
- 日报生成日期作为来源信息展示。
- 收藏数据使用统一收藏库，不按每天拆文件夹或文件。
- 网站是主要操作入口。
- GitHub 是主同步存储，当前允许公开。
- SecondBrain 是长期沉淀视图，不再承担主要交互。
- 链接旁不常驻星星，改用 hover 或右键浮层。
- 段落通过选中后浮层或右键菜单收藏。
- 整篇报告按钮放在标题旁。
